TimeStamp MicroService API

A simple microservice that fulfills the following qualifications (user stories):

  1. I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016).
  2. If it does, it returns both the Unix timestamp and the natural language form of that date.
  3. If it does not contain a date or Unix timestamp, it returns null for those properties.

Example Usage

Input:

https://fcc-ts-micro.herokuapp.com/january%201%202016 https://fcc-ts-micro.herokuapp.com/1451606400

Output:

{ "unix" : 1451624400, "natural" : "January 1st, 2016" }

Input:

https://fcc-ts-micro.herokuapp.com/notATimeStamp

Output:

{ "unix" : null, "natural" : null }

**Remember to put in spaces when passing in natural language dates, as you may get unexpected output otherwise**

Input:

https://fcc-ts-micro.herokuapp.com/january12016

Output:

{ "unix" : 317021126400, "natural" : "January 1st, 12016" }
